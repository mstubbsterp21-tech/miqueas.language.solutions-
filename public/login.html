Interpreter login without touching the live site.

This branch adds a reviewable interpreter portal concept for MLS. It is intentionally lightweight so the experience can be tested before connecting real authentication, document storage, or database records.

1
Centralized onboarding

Interpreters can eventually manage profile details, credentials, rates, availability, and required documents in one place.

2
Confidentiality-aware by design

This prototype avoids storing assignment, consumer, medical, legal, or school-specific details in the front end.

3
Ready for real auth later

The next step would be Clerk or Supabase Auth plus protected storage rules, not homemade password handling.
